CONFIGURATION PARAMETERS (Inline Descriptions + Examples)
==============================================================

CUSTOMERS
---------
customers.total_customers - Total number of customers to generate. Example: 1500000
customers.pct_india - Percentage of customers from India. Example: 1
customers.pct_us - Percentage of customers from the US. Example: 60
customers.pct_eu - Percentage of customers from Europe. Example: 39
customers.pct_org - Percentage of customers that are organizations. Example: 5
customers.seed - Random seed for reproducibility. Example: 42
customers.names_folder - Path to folder containing name lists. Example: ./data/customer_names

customers.geography_source.path - Path to geography parquet used to map customers. Example: ./data/.../geography_source.parquet
customers.geography_source.continent - Continent filter or 'All'. Example: All
customers.geography_source.max_geos - Max geography rows to load. Example: 1000


PROMOTIONS
----------
promotions.years - List of years to generate promotions for. Example: [2021, 2022, 2023]
promotions.num_seasonal - Number of seasonal promotions/year. Example: 20
promotions.num_clearance - Number of clearance promotions/year. Example: 8
promotions.num_limited - Number of limited-time promotions/year. Example: 12
promotions.seed - Random seed for promotions. Example: 42


STORES
------
stores.geography_path - Path to geographic dimension used for store placement. Example: ./data/parquet_dims/geography.parquet
stores.num_stores - Number of stores to generate. Example: 500
stores.opening_start - Earliest opening date. Example: 1995-01-01
stores.opening_end - Latest opening date. Example: 2020-12-31
stores.closing_end - Latest closing date. Example: 2025-12-31
stores.seed - Random seed. Example: 42


DATES
-----
dates.start_date - Start date of date dimension. Example: 2021-01-01
dates.end_date - End date of date dimension. Example: 2026-12-31
dates.fiscal_month_offset - Fiscal year start offset. Example: 5


SALES
-----
sales.parquet_folder - Folder containing generated parquet dimensions. Example: ./data/parquet_dims
sales.out_folder - Folder where sales chunks and final outputs are stored. Example: ./data/fact_out

sales.total_rows - Total sales rows to generate. Example: 1000000
sales.chunk_size - Number of rows to generate per chunk. Example: 500000

sales.start_date - Start date for sales generation. Example: 2021-01-01
sales.end_date - End date for sales generation. Example: 2025-10-31

sales.file_format - Output format (csv, parquet, delta). Example: csv
sales.write_delta - Whether to write delta files. Example: false
sales.delta_output_folder - Folder for delta output. Example: ./data/fact_out/delta

sales.merge_parquet - Merge chunks into one parquet file. Example: true
sales.merged_file - Name of merged parquet file. Example: sales.parquet
sales.delete_chunks - Delete chunk files after merge. Example: true

sales.row_group_size - Parquet row group size. Example: 5000000
sales.compression - Compression type. Example: snappy

sales.heavy_pct - % of heavy/large orders. Example: 5
sales.heavy_mult - Multiplier applied to heavy orders. Example: 5
sales.seed - Random seed. Example: 42

sales.workers - Number of worker processes (null = auto). Example: null
sales.write_pyarrow - Write parquet using PyArrow. Example: true
sales.tune_chunk - Auto-tune chunk size. Example: false

sales.skip_order_cols - Whether to skip order-level columns. Example: true


EXCHANGE RATES
--------------
exchange_rates.currencies - List of currencies to generate rates for. Example: ["USD","EUR","INR"]
exchange_rates.base_currency - Base currency. Example: USD
exchange_rates.start_date - Start date of exchange rate history. Example: 2021-01-01
exchange_rates.end_date - End date of exchange rate history. Example: 2025-10-31
exchange_rates.volatility - Level of fluctuation. Example: 0.02
exchange_rates.seed - Random seed. Example: 42
